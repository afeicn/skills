# 标书助手 Workflows

## 一、招标机会识别

目标：将招标公告、链接或截图转化为结构化机会摘要，并提醒关键时间。

```text
用户发送公告/链接/截图
  -> B01 接收招标机会
  -> B02 提取项目名称、采购人、预算、报名和投标截止时间
  -> 判断行业、区域、预算和业务相关性
  -> 生成机会摘要
  -> C14 设置报名、答疑、保证金和投标截止提醒
```

输出：机会摘要、关键时间表、初步推进建议、需补充文件。

## 二、投标可行性评估

目标：判断是否建议进入投标准备，并列出负责人必须确认的事项。

```text
读取机会摘要、招标文件和公司素材
  -> B04 提取资格条件
  -> B08 匹配公司资质、业绩、人员和交付能力
  -> B07 判断预算、周期、区域、竞争和履约风险
  -> 标注报价、商务承诺和最终投标决策的人工确认点
```

输出：建议推进 / 谨慎推进 / 不建议推进、缺口清单、风险等级、负责人确认事项。

## 三、招标文件解析

目标：提取招标文件中的资格条件、评分标准、提交要求和废标风险。

```text
上传招标文件
  -> C05 解析文档结构
  -> B03 提取章节和关键要求
  -> B04 提取资格条件
  -> B05 提取评分标准
  -> B06 提取提交格式、签章、份数、平台和保证金要求
  -> B15 生成需要澄清的问题
  -> 生成招标文件解析报告
```

输出：项目摘要、资格条件、评分标准、提交要求、废标条款、澄清问题。

## 四、原材料数字化

目标：把招标文件、作业设计、样例标书和知识库转为可全文检索材料。

```text
接收招标文件、作业设计、样例标书和知识库
  -> B17 检查文件是否已转换为可检索文本
  -> 抽取评分表、技术参数表、作业面积、服务期限和关键数字
  -> 标记扫描件、断裂表格、缺页、图片文字和无法检索段落
  -> 生成原材料索引和补转换清单
```

输出：原材料索引、关键数字清单、缺失/破损段落、需人工补录内容。

## 五、评分模型建立

目标：建立“这张标书凭什么得分”的模型。

```text
读取评分标准、资格条件和提交要求
  -> B18 建立评分模块、分值、性质和响应策略
  -> 区分实质性要求和评分项
  -> 识别客观分证明材料
  -> B19 生成实质性要求无负偏离索引
  -> 生成评分点索引总表和服务承诺证据链目录
```

输出：评分模型表、客观分材料清单、无负偏离索引、证据链目录。

## 六、投标任务拆解

目标：把招标要求拆解为可执行的投标协同任务。

```text
读取解析报告、评分模型和无负偏离索引
  -> B09 拆分商务、技术、报价、资质、盖章和上传任务
  -> 按评分项和废标风险标注优先级
  -> 分配责任人和截止时间
  -> C14 生成飞书待办和节点提醒
```

输出：投标任务清单、里程碑、责任人、截止时间、风险优先级。

## 七、服务方案结构设计

目标：在动笔前完成服务方案骨架、章节定位和篇幅分配。

```text
读取评分模型、作业设计和历史标书
  -> B20 对比样例标书，识别可借鉴元素和不可继承内容
  -> B21 生成十二章服务方案结构
  -> 在正文前部设置三张索引表
  -> 为每章生成“本章响应评分项”“本章写作定位”“本章成果资料”
  -> 标注重点章节篇幅和材料需求
```

输出：十二章服务方案结构、三张索引表、章节写作定位、篇幅分配、素材需求。

## 八、标书撰写辅助

目标：基于评分模型、作业设计和知识库生成项目化章节初稿。

```text
读取服务方案结构和作业设计
  -> B22 按章节扩写正文
  -> 将技术参数、面积、树种、苗木规格、施肥标准和工期写入对应章节
  -> 展开专项实施细则、SOP、逐月计划、风险预案和验收资料包
  -> 标记需人工补充的案例、证书、报价和承诺
```

输出：章节初稿、需补充材料、证据链缺口、人工确认事项。

## 九、表达净化与图表增强

目标：提升正式投标文件质感，并增加可核验的图表表达。

```text
读取章节初稿
  -> B23 检查 AI 腔、内部评分策略词、夸张词和不适合政府采购文件的表达
  -> 输出替换建议和仍需人工复核的句子
  -> B24 生成图表占位目录
  -> 为技术路线图、工序流程图、进度横道图、组织架构图和响应流程图生成占位说明
```

输出：表达净化报告、替换建议、图表目录、图表占位清单。

## 十、标书审查

目标：提交前检查缺项、偏离、格式、一致性、样例残留和废标风险。

```text
上传标书草稿或最终稿
  -> B12 检查商务材料缺项
  -> B13 检查格式、签章、页码、顺序和提交要求
  -> B14 检查技术/商务响应偏离和前后不一致
  -> B25 检查项目名称、编号、采购人、地点、服务期限和关键数字统一
  -> 检索样例标书残留和 AI 腔禁词
  -> 输出提交前一致性检查报告
```

输出：审查结论、致命风险、一般问题、整改建议、最终确认清单。

## 十一、投标复盘

目标：投标结束后沉淀经验、素材和可复用方法。

```text
输入投标结果、过程材料和审查报告
  -> B16 汇总关键动作和问题
  -> 分析中标或未中标原因
  -> 沉淀评分模型、优秀章节、风险清单和复用素材
  -> 更新知识库和模板库
```

输出：复盘报告、可复用素材、模板更新建议、下次投标改进清单。
