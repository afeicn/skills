# 招投标助手（Bid-Assistant）

> 版本：0.1.0  
> 当前目标：跑通“上传招标文件/作业设计 -> 建立评分模型 -> 生成服务方案结构 -> 输出章节初稿 -> 完成一致性审查”的 P0 闭环。

## 1. 员工定位

标书助手是面向市场、商务、方案、标书和管理层的岗位型数字员工。它覆盖从机会判断、评分模型、服务方案结构设计、章节写作到提交前质检的完整流程，特别适合政府采购类技术服务项目。

| 场景包 | 目标 |
| --- | --- |
| 招标机会识别 | 接收公告或链接，生成机会摘要和关键时间提醒 |
| 投标可行性评估 | 判断资质、业绩、预算、交付和竞争风险 |
| 招标文件解析 | 提取资格条件、评分标准、提交格式、废标风险 |
| 原材料数字化 | 将招标文件、作业设计、样例标书和知识库转成可检索材料 |
| 评分模型建立 | 区分实质性要求、评分项、客观分材料和响应章节 |
| 投标任务拆解 | 生成材料清单、责任人、截止时间和协同节点 |
| 服务方案结构设计 | 生成三张索引表、十二章结构、章节定位和篇幅分配 |
| 标书撰写辅助 | 根据评分项、作业设计和知识库生成章节初稿 |
| 表达净化与图表增强 | 清理 AI 腔、内部策略词，并生成图表占位目录 |
| 标书审查 | 检查缺项、格式、偏离、关键数字、样例残留和废标风险 |
| 投标复盘 | 总结中标/未中标原因，沉淀经验和素材 |

## 3. P0 演示闭环

```text
用户在飞书上传招标公告或招标文件
  -> 标书助手识别项目基本信息
  -> 提取资格条件和评分标准
  -> 建立评分模型和无负偏离索引
  -> 生成投标任务清单
  -> 生成服务方案十二章结构和章节初稿
  -> 输出表达净化和一致性审查意见
```

## 4. 关键边界

- 标书助手可以生成分析、清单、目录、初稿和审查意见。
- 标书助手不能替代投标负责人作最终投标决策。
- 报价、承诺、盖章、资质真实性、最终投标文件必须由人工确认。
- 涉及客户名称、金额、资质、过往业绩时，应按权限展示和归档。
- 正式投标文件中不得保留内部评分策略、夸张词和样例项目残留。
