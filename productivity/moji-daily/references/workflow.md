# Daily Task Tracker — 详细工作流

本文档供已触发 daily-task-tracker skill 的 Agent 参考，涵盖三种 Cron 触发场景和用户主动交互场景。

---

## 场景 A：早间 Review（09:00 Cron）

**目标**：确定用户当天的任务清单，写入 `tasks/YYYY-MM-DD.md`。

### 步骤
1. **检查文件是否已存在**：`ls tasks/YYYY-MM-DD.md`
   - 如果不存在 → 从 asset template 复制创建
   - 如果存在且有内容 → 先确认用户是否需要继续沿用或调整

2. **询问用户**（简洁版，不要列长问题列表）：
   > "早晨 ☀️ 今天有什么任务要安排？最高优先级是什么？"

3. **写入文件**：
   - 最高优任务写在 `🔥 今日最高优` 下
   - 其余写在 `今日待办` 下
   - 标注时间提醒用 `⏰ HH:MM 描述`
   - 标注未来日期用 `⏰ （MM/DD）描述`

---

## 场景 B：午间跟进（12:00 Cron）

**目标**：检查进度，适时提醒未完成事项。

### 步骤
1. **读取今天的任务文件**
2. **询问用户**：
   > "中午好，上午进展怎么样？XXX 做完了吗？有没有遇到问题？"

3. **更新文件**：
   - 已完成项：`- [ ]` → `- [x]`，加 ✅
   - 新增任务：追加到 `今日待办`
   - 更新 `⏰` 提醒事项的状态

---

## 场景 C：晚间总结（18:00 Cron）

**目标**：汇总今日完成情况，用户确认后滚动未完成项到次日。

### 步骤
1. **读取今天的任务文件**
2. **汇总状态**：统计已完成 vs 未完成，形成简报
3. **询问用户确认**：
   > "今天总结如下：\n✅ 完成 X 项：清单\n⏳ 未完成 Y 项：清单\n确认没问题？我把未完成的滚到明天。"

4. **用户确认后**（标记 `- [x] 已确认`）：
   - 创建 `tasks/YYYY-MM-DD.md`（明天）
   - 把未完成任务复制到明天的文件
   - 移入明天的提醒事项（`⏰` 标注）
   - 保留备注信息

5. **如果用户未确认**：保持现状，明天早上再问

---

## 场景 D：用户主动交互（随时）

### D1：用户说「XXX 完成了/做好了/做完了」

1. 读取今天的任务文件
2. 找到匹配的任务项，`- [ ]` → `- [x]`，加 ✅
3. 如果在 `今日待办` 已标完成，移到 `晚间总结 > 今日完成`
4. 回复确认

### D2：用户说「加一个任务 XXX」

1. 读取今天的任务文件
2. 在 `今日待办` 追加新项：`- [ ] **任务描述**`
3. 确认已添加

### D3：用户说「取消/删除 XXX」

1. 读取今天的任务文件
2. 将对应项改为：`- ~~任务描述~~` 或移到 `推迟/不做`
3. 确认已处理

### D4：用户说「把 XXX 改到明天/下周」

1. 读取今天的任务文件
2. 将对应项标记为推迟（加 ⏰ 日期标注）
3. 如果是当日滚动场景，注明推迟日期

---

## 滚动规则

### 格式约定
- `- [ ]` = 待办
- `- [x]` = 已完成（加 ✅ 表示用户确认完成）
- `⏰ HH:MM` = 当天时间提醒
- `⏰ （MM/DD）` = 未来日期事件
- `⚠️` = 需要注意的事项
- `🚨` = 最高优先级


### 文件管理
- 文件位置：`tasks/YYYY-MM-DD.md`
- 每天独立文件，零丢失
- 永不删除旧文件
- 跨日滚动时保留备注/反思内容
