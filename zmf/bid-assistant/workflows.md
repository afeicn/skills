# 标书助手 Workflows

## 一、招标机会识别

目标：将招标公告、链接或截图转化为结构化机会摘要。

```text
用户发送公告/链接/截图
  -> 接收文件或文本
  -> 提取项目名称、采购人、预算、截止时间
  -> 判断行业和业务相关性
  -> 生成机会摘要
  -> 设置关键时间提醒
```

## 二、投标可行性评估

目标：判断是否建议进入投标准备。

```text
读取机会摘要和招标文件
  -> 提取资格条件
  -> 匹配公司资质、业绩、人员和交付能力
  -> 判断预算、周期和竞争风险
  -> 输出建议推进/谨慎推进/不建议推进
```

## 三、招标文件解析

目标：提取招标文件中的关键要求。

```text
上传招标文件
  -> 解析文档结构
  -> 提取资格条件
  -> 提取评分标准
  -> 提取提交格式和材料要求
  -> 提取废标条款和风险项
  -> 生成招标文件解析报告
```

## 四、投标任务拆解

目标：把招标要求拆解为可执行任务清单。

```text
读取解析报告
  -> 生成材料清单
  -> 拆分商务、技术、报价、资质、盖章等任务
  -> 分配责任人
  -> 标注截止时间
  -> 生成飞书待办
```

## 五、标书撰写辅助

目标：基于模板和公司素材生成响应目录和章节初稿。

```text
读取招标要求和响应目录
  -> 匹配公司资质、案例和方案素材
  -> 生成响应目录
  -> 生成章节初稿
  -> 标记需人工补充内容
```

## 六、标书审查

目标：在提交前检查缺项、偏离、格式和一致性问题。

```text
上传标书草稿
  -> 对照招标文件要求
  -> 检查缺项
  -> 检查格式和签章要求
  -> 检查响应偏离
  -> 检查前后不一致
  -> 输出审查报告
```

## 七、投标复盘

目标：投标结束后沉淀经验和复用素材。

```text
输入投标结果和过程材料
  -> 汇总关键动作
  -> 分析中标或未中标原因
  -> 提炼可复用素材
  -> 生成复盘报告
```
