# 标书助手数字员工

> 版本：2026-05-26  
> 部署建议：电脑 2，飞书入口“小招”，Hermes Agent 员工 ID：`bid_assistant`  
> 当前目标：先跑通“上传招标文件 -> 标书助手解析 -> 提取资格与评分项 -> 生成投标任务清单 -> 生成响应目录 -> 输出审查意见”的 P0 闭环。

## 1. 员工定位

标书助手是面向市场、商务、方案、标书和管理层的岗位型数字员工。它不只是写标书，而是覆盖从机会判断到投标复盘的完整流程。

| 场景包 | 目标 |
| --- | --- |
| 招标机会识别 | 接收公告或链接，生成机会摘要和关键时间提醒 |
| 投标可行性评估 | 判断资质、业绩、预算、交付和竞争风险 |
| 招标文件解析 | 提取资格条件、评分标准、提交格式、废标风险 |
| 投标任务拆解 | 生成材料清单、责任人、截止时间和协同节点 |
| 标书撰写辅助 | 根据模板和素材生成响应目录和章节初稿 |
| 标书审查 | 检查缺项、格式、偏离、前后不一致和风险 |
| 投标复盘 | 总结中标/未中标原因，沉淀经验和素材 |

## 3. P0 演示闭环

```text
用户在飞书上传招标公告或招标文件
  -> 标书助手识别项目基本信息
  -> 提取资格条件和评分标准
  -> 判断是否建议投标
  -> 生成投标任务清单
  -> 生成响应目录
  -> 输出缺项和风险审查意见
```

## 4. 关键边界

- 标书助手可以生成分析、清单、目录、初稿和审查意见。
- 标书助手不能替代投标负责人作最终投标决策。
- 报价、承诺、盖章、资质真实性、最终投标文件必须由人工确认。
- 涉及客户名称、金额、资质、过往业绩时，应按权限展示和归档。
