# 标书助手 Runbook

## 一、部署前准备

1. 确认电脑 2 可运行 Hermes Agent。
2. 确认飞书机器人“小招”已创建。
3. 准备示范招标文件、历史标书模板、公司资质和项目案例。
4. 确认 `employee.yaml` 中的 `hermes_agent_id` 与实际配置一致。

## 二、启动检查

```text
1. 启动 Hermes Agent
2. 加载 employee.yaml
3. 加载 bid_skills.yaml
4. 连接飞书机器人
5. 执行 test_cases/p0_demo_cases.md 中的样例 1
6. 检查日志是否写入
```

## 三、演示前检查

- 飞书账号已登录。
- 标书助手对话窗口已置顶。
- 演示招标文件已准备。
- 录屏文件可直接播放。
- 输出模板可正常渲染。

## 四、故障处理

| 问题 | 处理方式 |
| --- | --- |
| 文件解析失败 | 切换为预生成解析结果 |
| 文档太大 | 使用节选版演示文件 |
| 输出不稳定 | 使用固定提示词和录屏 |
| 风险项缺引用 | 切换为人工标注版演示结果 |

## 五、生产化待办

- 对接招标公告来源或人工转发入口。
- 整理公司资质库、案例库和历史标书库。
- 建立评分项和废标风险检查规则。
- 对接飞书待办，生成投标任务协同。
- 增加投标复盘和素材沉淀流程。
