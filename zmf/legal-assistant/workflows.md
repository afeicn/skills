# 法务助手 Workflows

## 一、合同接收

目标：接收合同并识别类型、主体、金额、期限和业务场景。

子角色：合同受理员。

主要调用：L01_contract_intake、L02_contract_key_extract。

```text
用户通过飞书提交任务、文字或附件
  -> C01 飞书消息接收与路由
  -> C03 用户身份与权限识别
  -> C04 任务意图识别
  -> C05/C07 解析材料并抽取结构化字段
  -> C08 检索相关知识库、模板、SOP 和历史材料
  -> 调用 L01_contract_intake + L02_contract_key_extract
  -> C09 按规则校验风险、缺口和需确认事项
  -> C10/C11 生成结构化结果、表格或文档草稿
  -> C13 提交人工确认或审批
  -> C15/C16 记录日志并归档
```

输出：结构化结果、风险提示、下一步动作、需人工确认事项和归档记录。

人工确认：涉及最终对外提交、金额、合同、承诺、敏感信息或高风险判断时，必须进入人工确认。
