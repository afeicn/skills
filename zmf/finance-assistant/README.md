# 财务助手数字员工

> 版本：2026-05-26  
> 部署建议：电脑 1，飞书入口“小财”，Hermes Agent 员工 ID：`finance_assistant`  
> 当前目标：先跑通“飞书发送发票 -> 财务助手识别 -> 报销申请受理 -> 报销审核 -> 主管审批 -> 台账归档”的 P0 闭环。

## 1. 员工定位

财务助手是一个财务岗位型数字员工，不是单一报销工具。它内部包含多个业务场景包：

| 场景包 | 目标 |
| --- | --- |
| 报销申请受理 | 接收员工发票和报销说明，生成报销单草稿 |
| 报销单审核 | 对发票、附件、费用标准、审批权限进行审核 |
| 审批流转 | 生成飞书审批卡片，提交主管或财务复核 |
| 制度问答 | 回答报销流程、差旅标准、附件要求等问题 |
| 台账与分析 | 更新报销台账，输出费用统计和异常提示 |

## 3. P0 演示闭环

```text
员工在飞书发送发票图片 + 报销说明
  -> 财务助手接收文件
  -> 识别发票字段
  -> 判断费用类型
  -> 检查附件完整性和重复报销
  -> 生成报销单草稿
  -> 输出财务审核意见
  -> 提交主管审批卡片
  -> 更新报销台账
```

## 4. 关键边界

- 财务助手可以生成草稿、审核建议、风险提示和审批卡片。
- 财务助手不能替代财务人员最终确认。
- 发票验真生产版需要真实税务或第三方接口；演示版可使用模拟验真结果。
- 涉及金额、银行账户、身份证、薪酬、合同金额等敏感信息时必须脱敏展示。
