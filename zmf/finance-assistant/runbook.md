# 财务助手 Runbook

## 一、部署前准备

1. 确认电脑 1 可运行 Hermes Agent。
2. 确认飞书机器人“财务助手”已创建。
3. 确认机器人具备接收消息、接收附件、发送卡片的权限。
4. 准备演示发票、报销制度、报销单模板和历史台账样例。
5. 确认 `employee.yaml` 中的 `hermes_agent_id` 与实际配置一致。

## 二、启动检查

```text
1. 启动 Hermes Agent
2. 加载 employee.yaml
3. 加载 finance_skills.yaml
4. 连接飞书机器人
5. 执行 test_cases/p0_demo_cases.md 中的样例 1
6. 检查日志是否写入
```

## 三、演示前检查

- 飞书账号已登录。
- 财务助手对话窗口已置顶。
- 演示发票在本地和飞书中都有备份。
- 录屏文件可直接播放。
- 模拟验真结果已准备。
- 审批卡片接收人使用演示账号，不要发送给真实主管。

## 四、故障处理

| 问题 | 处理方式 |
| --- | --- |
| 飞书收不到文件 | 改用本地演示文件，并说明演示使用本地文件 |
| OCR 失败 | 切换预生成识别结果 |
| 验真接口不可用 | 使用 demo_mode 模拟验真 |
| 审批卡片发送失败 | 展示 markdown 版审批卡片 |
| 模型输出不稳定 | 使用固定预设问题和录屏 |

## 五、生产化待办

- 对接真实发票验真接口。
- 对接真实飞书审批流程。
- 对接财务台账或财务系统。
- 完成财务制度结构化。
- 配置用户权限和敏感信息脱敏。
- 增加异常报销人工复核队列。
