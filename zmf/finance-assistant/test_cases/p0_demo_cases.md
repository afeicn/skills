# 财务助手 P0 演示测试样例

## 样例 1：标准交通费报销

用户飞书输入：

```text
@财务助手 我报销一下今天去客户那边的交通费，发票见附件，项目归属：客户拜访项目。
```

附件：

- demo_invoice_transport.png

期望流程：

1. 识别发票信息。
2. 判断费用类型为交通费。
3. 检查是否缺少附件。
4. 检查是否疑似重复报销。
5. 生成报销单草稿。
6. 生成审核意见。
7. 生成主管审批卡片。

期望输出摘要：

```text
处理结论：通过建议，待员工确认后提交主管审批。
风险提示：发票验真为演示模式结果，正式报销需财务复核。
```

## 样例 2：附件缺失

用户飞书输入：

```text
@财务助手 报销昨晚客户接待餐费，发票已上传。
```

期望输出：

```text
处理结论：需补充材料。
需要补充：接待事由、参与人员、审批前置记录或项目归属。
```

## 样例 3：疑似重复报销

用户飞书输入：

```text
@财务助手 帮我提交这张办公用品发票。
```

期望输出：

```text
处理结论：需人工复核。
原因：历史台账中存在相同发票号码或相同金额、日期、销售方记录。
```
